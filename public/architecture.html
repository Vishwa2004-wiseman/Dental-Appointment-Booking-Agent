🦷 Dental Booking Agent — How it works

A patient calls → an AI books the appointment → it lands in Google Calendar, sends an SMS, and is saved in the database.

1

📞 Patient calls the clinic

A real phone call comes in. The clinic doesn't need a human receptionist to pick up.

↓
2

🤖 VAPI voice AI answers

VAPI turns the caller's speech into text and sends it to my server as a webhook (an HTTP POST) after each thing they say.

POST /webhooks/vapi
↓
3

🧠 My server tracks the conversation

A simple state machine moves through fixed steps: name → service → date & time → confirm. It always knows which step the call is on.

src/state/flow.js
↓
4

🔥 Every step is saved in the database

State is stored in Firestore, keyed by the call ID — not in memory. So if the server restarts, nothing is lost, and each caller stays separate.

src/services/firestore.js
↓
5

✅ On “yes”, two real actions happen

📅 Google Calendar — creates the appointment event
💬 Twilio — sends an SMS confirmation
calendar.js  ·  sms.js
↓
6

🖥️ Staff see everything on a dashboard

A web dashboard and an admin API read from the database to show all bookings and every call's history.

public/index.html  ·  /admin, /api
Built with Node.js + Express · Firestore · Google Calendar API · Twilio · deployed on Railway.
Stateless servers + a clinicId on every record → scales from one clinic to a thousand.
